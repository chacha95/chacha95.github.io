## Frequentist Inference

### Estimation

Statistics의 주요 목표 중 하나는 알 수 없는 population의 parameter(모수)를 추정하는 것 입니다. 이러한 parameter를 근사하기 위해 estimator를 사용합니다. 여기서 말하는 parameter의 대표는 바로 mean과 varaince 입니다.

Frequentist 입장에서의 estimator는 크게 2가지 방식으로 나뉩니다.

**Point Estimation(점 추정)**

Population으로 부터 sample을 추출하고, 추출된 sample의 통계량(mean, variance)를 통해 population의 parameter를 근사 하는 추정 방식입니다.



**Interval Estimation(구간 추정)**

Point estimator와 달리 interval estimation은 가능한 값 범위를 지정하여 parameter를 추정합니다. 즉 point estimation과는 다르게 존재할 구간을 확률적으로 추정하는 방식입니다.

대표적인 방식이 바로 confidence interval(신뢰 구간) 방식입니다. Confidence interval이란 이 구간 내에 실제 population이 존재할 것으로 예측되는 구간으로 정의가 되며, 90%, 95%, 99% confidence interval 등 다양한 정도의 구간 추정이 가능합니다. 이 중 가장 널리 사용되는 것은 95% confidence interval 입니다. 이는 예측된 구간 내에 실제 population mean이 있을 가능성이 95%라고 신뢰할 수 있는 구간의 의미입니다.



<br>

### The Bootstrap

Frequentist inference의 대부분은 "좋은" estimator의 사용에 중점을 둡니다. 

그렇다면 어떻게 해야 더 좋은 estimator를 만들까요? 그 중 대표적인 방법론이 바로 bootstrap입니다. Bootstrap이라고하는 기술은 resampling을 통해 estimator의 population을 더 정확히 추정하는 방법입니다. 

Bootstrap은 statistic과 machine learning에서의 정의가 약간 다르며 다음과 같습니다.

Statistic에서의 bootstrap은 resampling을 통해 좀 더 정확한 mean/variance를 추정하는 방법을 말합니다.

Machine learning에서는 resampling을 통해 학습 데이터를 늘리는 방식을 말하며, 이를 통해 데이터 셋 내에 데이터 분포가 고르지 않을 때 유용합니다. Bootstrap aggregating의 줄임말인 **bagging**이라고도 불립니다.

## Bayesian Inference

### Bayes' Rule

두 random variable의 prior probability와 posterior probability 사이의 관계를 나타내는 정리입니다.

Bayes' Rule에 대해 이해하기 위해선 conditional probability에 대해 이해가 필요합니다.



**Conditional Probability(조건부 확률)**

Conditional probability는 전체 sample space가 아닌 특정 event만 주목하는 확률입니다.

아래 그림의 예시에서 전체 sample space **S**에서 sub space **B**만을 주목한 뒤, **B**안에서의 **A and B**일 확률을 찾는것입니다.

> 대표적인 예시

<center><img src="https://user-images.githubusercontent.com/31475037/65304509-c7dc9480-dbbb-11e9-82cb-efca742e925a.PNG"></center>
> Conditional Probability 수식

$$
P(A|B) = \frac {P(A\cap B)} {P(B)}
$$



**Joint Probability(결합 확률)**

Joint probability란 **두 event이 순차적으로 연속해서 일어날 확률** 혹은 **시간적으로 동시에 일어날 확률**을 의미합니다. 일반적으로 joint probability의 계산은 두 확률을 chain rule 처럼 곱하면 됩니다. 

> Joint Probability

$$
 P(A \cap B) = P(A) \cdot P(B | A) = P(B) \cdot P(A|B)
$$



이 때 joint probability의 chain rule을 이해하는 두가지 관점이 있습니다.

1. 연속된 또는 연결된 두 event의 순차적 흐름으로 보는 관점

   > 순차적인 관점

<center><img src="https://user-images.githubusercontent.com/31475037/65305173-78976380-dbbd-11e9-8eb2-ac4197216ffb.png"></center>
2. 특정 관심 event이 일어날 **likelihood(가능성)**의 관점

   일종의 역발상으로 이미 알고 있는 어떤 event의 수학적 **prior probability(사전확률)**과 경험상 얻어낸 **likelihood**의 곱으로 **posterior probability(사후 확률)**를 추정하는 것입니다. Likelihood는 계산값이 아니고 관측한 데이터이며, 일반적으로 문제 상황에 따라 적절하게 주어집니다.



**Marginal Probability(주변 확률)**

Joint probability가 주어진 상태에서 오직 한가지 random variable의 확률 분포를 알고 싶을 때 사용하는 방법입니다. 두 개 이상의 확률로 결합된 결합확률에서 하나의 확률만 선택하여 나타낸 확률입니다. 주로 summation을 통해 나머지 확률을 없앱니다.



**Bayes' Rule(베이즈 정리)**

Bayes' Rule은 posterior probability(posterior)를 설명하기 위해 만들어진 정리입니다. 즉 ,결과를 전제로 한 확률을 원인을 전제로 한 확률로 바꾸어 표현하고 있는 것이죠. 이렇게 베이즈 정리는 앞서 일어나는 event을 전제로 하는 conditional probability와 prior probability을 이용해 posterior probability를 구하는 법을 설명하고 있는 정리입니다.

> Bayes' Rule

$$
P(B |A) = \frac {P(A|B)P(A)}{P(B)}
$$

> Posterior Probability

$$
P(A|B)
$$



> Prior Probability

$$
P(A): \ A의 \ prior \ probability \\
P(B): \ B의 \ prior \ probability
$$



먼저 Bayes' Rule의 수식을 이해하기 위해서는 posterior의 의미를 알아야 합니다.

Conditional probability는 **어떤 event가 만들어 놓은 상황에서, 그 event가 일어난 후 앞으로 일어나게 될 다른 event의 가능성을 구하는 것**입니다. 예를 들어 검은색 공 3개와 흰색 공 3개가 들어있는 상자에서 검은색 공 1개를 뽑은 다음, 뽑은 공을 다시 집어 넣지 않은 상태에서 다시 공 1개를 뽑는 시행을 생각해보겠습니다. 이때, 2번째 뽑은 공의 색이 검은색일 가능성은 어떻게 계산할 수 있을까요? (남은 5개의 공은 각각 뽑힐 가능성이 모두 같다고 가정하겠습니다.) **처음 공을 뽑을 때 이미 검은색 공을 뽑아 놓았기 때문에** 상자 안에는 검은색 공 2개와 흰색 공 3개가 남아있습니다. 따라서 2번째 뽑은 공의 색이 검은색이 될 확률은 0.4입니다.

하지만 나중에 일어나는 event을 전제로 하는 conditional probability인 posterior는 먼저 일어나는 event를 전제로 하는 conditional probability와는 의미와 목적이 서로 다릅니다. 나중에 일어나는 event가 만들어 놓은 결과를 바탕으로 그 후에 그보다 앞서 일어나는 event가 일어난다는 것은 시간적으로 불가능한 순서이기 때문입니다. 예를 들어 오후 3시에 일어나는 event을 바탕으로 같은 날 오후 1시에 다른 event가 발생하였다라고 말할 수는 없기 때문입니다. posterior의 의미를 이해하기 위해서 다음과 같은 상황을 생각해보겠습니다.



**Posterior란 원인들의 영향을 비교하는 것**

event B 또는 event C가 원인이 되어 일어나는 event A가 있습니다. 그리고 event A가 100번 일어날 때까지 매번 event B나 C 중 어떤 event의 원인이 되어 event A가 일어났는지 그 원인을 조사해보니 event B가 원인이 되어 event A가 일어난 것이 40번, event C가 원인이 되어 event A가 일어난 것이 60번이었습니다.

event A가 100번 일어나는 동안, event B가 원인이 된 것은 모두 40번이므로 event A가 일어났을 때, event B가 원인이 되었을 확률은 0.4 입니다.

> event A가 일어났을 때, event B가 원인일 확률은?

$$
P(B|A) = \frac {40} {100} = \frac{2}{5}
$$


event A가 100번 일어나는 동안, event C가 원인이 된 것은 모두 60번이므로 event A가 일어났을 때, event C가 원인이 되었을 확률은 0.6 입니다.

> event A가 일어났을 때, event C가 원인일 확률은?


$$
P(C|A) = \frac {60} {100} = \frac {3} {5}
$$



두 확률 모두 "event A가 일어났을 때"로 event A가 일어났다는 것을 전제하고 있습니다. 그리고 분모의 숫자 100은 모든 원인들의 수를 합한 것을 의미합니다. 

이를 해석해보면 **posterior는 event A가 일어났을 때, 다른 여러가지 원인들과 비교하여 event B가 얼마나  event A가 발생하는데 영향을 주었는지를 평가하는 것**이라 할 수 있습니다. 즉 posterior에서 나중에 일어나는 event A를 전제한다는 것은 event A가 일어나도록 한 원인들을 모두 찾고 그 원인들이 event A가 일어나는데 얼마나 영향을 주었는지를 비교하는 것을 뜻합니다.

그렇다면 event B가 event A가 일어나는데 얼만큼 영향을 주었는지는 어찌 계산 가능 할까요? 그건 바로 이 두 확률을 곱해주면 얻을 수 있습니다.

> Bayes' Rule 우변

$$
P(A|B) \cdot P(B)
$$

이후 , event A가 일어난 확률로 나눠주면 Bayes' Rule 수식이 완성됩니다.

> P(A)로 나눠줌

$$
P(B|A) = \frac {P(A|B) \cdot P(B)} {P(A)}
$$

## Frequentist and Bayesian

일반적인 확률의 개념은 실험을 반복적으로 (예를 들어, 카드를 뽑는 것) 했을 때 나오는 빈도를 확률로 표현합니다.

하지만 이 방법은 실험을 반복 할 수없는 경우에는 적용이 불가 합니다. 

예를들어, 의사가 환자를 진찰하고 병에 걸렸는지 아닌지 판단한다고 해봅시다. 의사가 환자를 진찰한 후 50%의 확률로 병에 걸렸을 거라고 말합니다. 이 경우 50%의 의미가 우리가 일반적으로 아는 확률인가요?

의사가 환자를 똑같은 상태로 복제 해 여러번 실험 할 수 없습니다. 동일한 증상이 있을 수 있지만, 환자의 상태는 다 다릅니다. 

이럴 땐, 확률이 믿음의 정도를 나타내는데, 1은 환자가 병에 걸렸다는 절대적 믿음을 나타내고 0은 환자가 병에 걸리지 않았다는 절대적인 믿음을 나타냅니다. 

전자인 실험을 반복적으로 해 확률을 도출하는 방식이 frequency statistic이고,  후자인 의사가 환자를 진찰해 확률적으로 결과를 말해주는 것이 bayesian statistic입니다.
