# DOM

DOM은 HTML, XML 문서의 프로그래밍 interface입니다. DOM은 문서의 구조화된 표현을 제공하며, 프로그래밍 언어가  DOM 구조에 접근할 수 있는 방법을 제공하여 문서 구조, 스타일, 내용 등을 변경할 수 있게 합니다. DOM은 프로그래밍 언어는 아니지만 , JS가 document들을 컨트롤 할 수 있게 하는 interface입니다.

DOM을 사용하기 위해 특별히 해야할 일은 없습니다. 각각의 브라우저는 자신만의 방법으로 DOM을 구현하였습니다.

### DOM Parser

모든 브라우저에는 HTML 문서를 DOM으로 구문 분석하는 DOM Parser가 있습니다. DOM parser는 HTML 페이지를 읽고 해당 데이터를 DOM을 구성하는 객체로 바꿔줍니다.

### DOM Manipulation

돔 조작은 웹 페이지를 수정하기 위해 매우 유용하지만, 문제점도 존재합니다. 트리에서 하나의 노드를 추가하거나 제거한다면, 전체 트리를 제거하는 경우가 생길 수도 있다는 점입니다. 이 작업은 리소스가 많이 필요한 작업입니다.

비슷한예로 하나의 노드 추가, 삭제로 인해 웹 페이지 전체 레이아웃이 영향을 받을시, 웹페이지의 일부 또는 전체가 다시 렌더링 될 수 있습니다. 이런 경우를 Reflow라 부릅니다.

### Browser's Workflow

1. **DOM tree 생성**: 브라우저가 HTML 파일을 수신하면 렌더 엔진은 해당 파일을 parsing하고 HTML 각 요소를 NODE로 만들어 DOM 트리 생성
2. **Render Tree 생성**: 인라인 스타일 구문과 외부 CSS 파일의 스타일이 구문 분석이 되며, DOM Tree의 node와 함께 Render Tree라는 다른 트리를 만드는데 사용
3. **Layout**: Render tree를 생성한 후에 "Layout" 프로세스를 거친다. Render Tree의 모든 node에는 화면에 표시되어야 하는 정확한 위치인 좌표가 제공
4. **Paining**: Render Tree가 통과되고 각 노드의 "paint()" 함수가 호출되어 궁극적으로 화면에 내용이 표시

위 과정에서 알 수 있듯이, DOM을 만들 때마다 Render Tree 생성부터 layout, painting까지 해야하기에 DOM 자체는 빠르지만, reflow가 너무 많습니다. 그렇기에 virtual dom이 나왔습니다.

### Virtual DOM

실제 DOM의 변경사항에 대해 DOM에서 수행해야 할 모든 변경 사항을 가상돔(virtual DOM)에서 수행한 다음 실제 DOM에 전달함으로써 위에서 언급한 계산 단계가 줄어듭니다. 여러 번의 변경사항이 있더라도 모든 변경 사항을 하나로 그룹화하여 한번만 수행합니다. 마지막으로 DOM 조작 자체를 포기함으로써 DOM을 수정하는 모든 부분 간의 동기화를 피할 수 있다.

## Chrome V8 engine

자바스크립트 엔진은 자바스크립트 코드를 실행하는 인터프리터를 말합니다. 표준적인 인터프리터로 구현될 수도 있고, JIT 컴파일러로도 구현될 수 있습니다.

고 수준의 언어는 기계어에 비해 많은 추상화가 이루어져있습니다. 아래 추상화의 수준에서 자바스크립트가 기계어로부터 얼마나 추상화 됐는지 알 수 있습니다. C/C++는 상대적으로 하드웨어 언어에 가깝고 다른 고수준 언어들보다 훨씬 빠릅니다.

> 다양한 종류의 언어들

![](https://user-images.githubusercontent.com/31475037/84244993-5ffe1d80-ab3f-11ea-9377-1f9ce8d42aa1.png)

V8 엔진은 구글이 제공하는 강력한 오픈소스 자바스크립트 엔진입니다. 자바스크립트 엔진이란 자바스크립트 코드를 cpu가 이해할 수 있는 낮은 수준의 언어 혹은 기계어로 변환해주는 역할을 합니다.

다양한 종류의 자바스크립트 엔진이 존재하지만, 모두 ECMAScript 표준을 따릅니다. ECMAScript는 스크립팅 언어를 위한 표준을 정의합니다.

크롬 V8엔진의 특성은 다음과 같습니다.

- V8엔진은 C++로 작성됐고, Chrome과 Node.js에서 사용됩니다.
- V8엔진은 ECMA-262에 기재된 ECMAScript를 구현했습니다.
- V8엔진은 standalone으로 동작할 수 있어서 우리는 자바스크립트 엔진을 C++ 프로그램에 내장시킬 수 있습니다.

## Ajax(Asynchronous JavaScript and XML)

Ajax는 JS와 XML을 이용한 비동기적 정보 교환 기법을 의미합니다.

> Ajax에 대한 설명

<center><img src="https://t1.daumcdn.net/cfile/tistory/992AEC505AF6EF6032"></center>

Ajax가 등장하기 이전에는 웹 브라우저가 정보를 요청하면 서버는 해당 정보를 포함한 페이지 전체를 전달해 주었습니다. 웹 브라우저는 서버로부터 받은 HTML을 브라우저 창에 렌더링하는 일을 했고 이 방법은 웹 브라우저에게도, 서버에게도 서로 불편했는데 브라우저는 매번 똑같은 레이아웃의 페이지를 처음부터 다시 렌더링해야 했고 서버 역시 매번 같은 페이지를 렌더링(이 경우에는 HTML 파일 생성)해야 해서 서로 부담이 되었습니다. 오래된 웹사이트 중 Ajax 기술을 사용하지 않는 사이트가 특유의 깜빡거림 현상이 생기는 이유가 바로, 매번 페이지를 싹 지우고 처음부터 다시 모든 것을 그려내기 때문입니다.

Ajax는 웹 페이지의 디자인 요소와 정보 요소를 분리합니다. 처음 사이트에 접속하면 웹 서버는 해당 페이지의 스타일과 레이아웃을 정의하는 정적인 HTML 파일과 CSS 파일 그리고 데이터를 어떻게 요청하면 되는지를 설명한 JavaScript 파일을 전달합니다. 브라우저는 일단 이 내용으로 페이지의 기본적인 골격을 구축해 놓습니다. 골격의 구축이 끝나면 브라우저는 전달받은 JavaScript를 실행해서 서버에 데이터만을 별도로 요청합니다. 이 때 서버에서 주는 응답은 순수한 데이터 객체 즉 XML 또는 JSON 데이터입니다. 브라우저가 이 데이터를 수신하면 미리 만들어 둔 틀 안에 적절한 방법으로 데이터를 끼워 넣습니다.

**과정**

1. 브라우저가 사이트에 접속하면 서버는 사이트의 기본 구조를 담은 '템플릿'을 전달
2. 브라우저는 수신받은 템플릿 HTML과 CSS를 해석해 화면의 기본 모양을 그림
3. 계속해서 서버는 데이터의 요청 방식과 수신받은 데이터를 어떻게 가공해야 하는지를 기술한 JavaScript 파일을 전달
4. 브라우저는 자바스크립트 파일을 해석해 파일에 기술된 방식대로 서버에 추가 데이터를 요청
5. 서버는 데이터를 응답
6. 브라우저는 수신한 데이터를 해석, 템플릿의 적절한 위치에 삽입
