# FastAPI란?

[FastAPI](https://fastapi.tiangolo.com/)는 현대적이고, 빠르며(고성능), 파이썬 표준 타입 힌트에 기초한 Python3.6+의 API를 빌드하기 위한 웹 프레임워크입니다.

> FastAPI

![](https://fastapi.tiangolo.com/img/logo-margin/logo-teal.png)



주요 특징으로:

- **빠름**: (Starlette과 Pydantic 덕분에) **NodeJS** 및 **Go**와 대등할 정도로 매우 높은 성능. [사용 가능한 가장 빠른 파이썬 프레임워크 중 하나](https://fastapi.tiangolo.com/ko/#performance).
- **빠른 코드 작성**: 약 200%에서 300%까지 기능 개발 속도 증가.
- **적은 버그**: 사람(개발자)에 의한 에러 약 40% 감소.
- **직관적**: 훌륭한 편집기 지원. 모든 곳에서 자동완성. 적은 디버깅 시간.
- **쉬움**: 쉽게 사용하고 배우도록 설계. 적은 문서 읽기 시간.
- **짧음**: 코드 중복 최소화. 각 매개변수 선언의 여러 기능. 적은 버그.
- **견고함**: 준비된 프로덕션 용 코드를 얻으세요
- **표준 기반**: API에 대한 (완전히 호환되는) 개방형 표준 기반: [OpenAPI](https://github.com/OAI/OpenAPI-Specification) (이전에 Swagger로 알려졌던) 및 [JSON 스키마](http://json-schema.org/).

<br>

# FastAPI의 구성 요소

FastAPI를 구성하는 요소는 크게 2가지로 볼 수 있습니다. 

**Uvicorn** 그리고 **Starlette**.

<br.

## Uvicorn (ASGI Server)

**Uvicorn**은 초고속 **ASGI** Web server입니다. **단일 프로세스**에서 uvloop기반 비동기 Python code를 실행합니다.

자세한 내용은 [이전 포스트 참조](https://chacha95.github.io/2021-01-30-python6/) 부탁드립니다.

<br>

## Starlette(Web Application Server)

Starlette는 비동기적으로 실행할 수있는 Web application server 입니다. Starlette는 Uvicorn을 사용하여 실행합니다. 

<br>

## FastAPI

FastAPI는 Starlette 위에서 많은 기능을 제공합니다. FastAPI를 사용하지 않고 Starlette만 사용했다면, 모든 데이터 유효성 검사 및 직렬화를 직접 구현해야합니다.이 때문에 최종 애플리케이션은 FastAPI를 사용하여 빌드하는 것과 동일한 오버 헤드를 갖습니다. 그리고 대부분의 경우, **데이터 유효성 검사 및 직렬화는 애플리케이션에서 작성되는 가장 많은 양의 코드입니다.**
따라서 FastAPI를 사용하면 개발 시간, 버그, 코드 줄을 절약 할 수 있으며 사용하지 않았을 때와 동일한 성능 (또는 더 나은 성능)을 얻을 수 있습니다.

<br>

# Uvicorn-Gunicorn-FastAPI(For Deployment)

Uvicorn-FastAPI에서 Uvicorn은 **single process**로만 작동합니다. 이 때문에 병렬성이 부족해져 여러 process를 돌려야 하는 상황에 대처할 수 없습니다. 때문에 Gunicorn을 통해 프로세스를 동시에 여러개를 늘려 병렬성을 확보해야합니다.

<br>

## Gunicorn

Gunicorn은 성숙하고 완전한 기능을 갖춘 **서버** 및 **프로세스** 관리자입니다. 

Python web application을 production 환경에 deployment 할 때 일반적인 세 가지 구성 요소는 다음과 같습니다.

- **Web server** (e.g. Nginx)
- **WSGI (web) server** (e.g. Gunicorn)
- **Web application server**(web app or web framwork) (e.g. Django)

Web server는 요청을 수락하고 도메인를 처리하며 https 연결을 처리합니다. WSGI server는 web server의 요청을 받아 web application server에 전달합니다. Web applicatin server는 단일 요청을 처리 할 수 있다는 것 외에는 아무것도 신경 쓰지 않습니다.

### Gunicorn is a WSGI server

Gunicorn은 다양한 web server와 상호 작용할 수 있도록 구축되었습니다. Gunicorn은 웹 서버와 웹 애플리케이션 사이에서 일어나는 모든 일을 처리합니다. 이렇게하면 웹 애플리케이션을 코딩 할 때 다음을위한 자체 솔루션을 찾을 필요가 없습니다.

- 여러 웹 서버와 통신
- 한 번에 많은 웹 요청에 반응하고 부하를 분산
- 실행중인 웹 애플리케이션의 여러 프로세스 유지

**WSGI (Web Server Gateway Interface)는 web server와 web application이 서로 통신 할 수 있도록하는 방법입니다.** 즉, 서버 측은 애플리케이션 측에서 제공하는 호출 가능한 객체를 호출합니다. 따라서 애플리케이션 내부 어딘가에 Gunicorn에서 호출 할 수있는 개체가 정의됩니다. 이 개체는 응용 프로그램에 요청 데이터를 전달하고 응답 데이터를 수신하는 데 사용됩니다. Gunicorn은 웹 애플리케이션의 여러 인스턴스를 실행하여 정상인지 확인하고 필요에 따라 다시 시작하여 해당 인스턴스에 수신 요청을 분산하고 웹 서버와 통신합니다.

<br>

## Uvicorn with Gunicorn

Uvicorn에는 Gunicorn worker class가 포함되어있어 Uvicorn의 모든 성능 이점과 함께 ASGI 애플리케이션을 실행할 수 있으며 Gunicorn의 모든 기능을 갖춘 프로세스 관리를 제공합니다. 이를 통해 즉시 worker 프로세스 수를 늘리거나 줄이고, worker 프로세스를 정상적으로 다시 시작합니다.

최종적으로 Gunicorn을 사용하여 Uvicorn을 관리하고 이를 통해 동시에 여러 프로세스를 실행할 수 있습니다.

[Uvicorn-Gunicorn-FastAPI official dockerfile repo](https://github.com/tiangolo/uvicorn-gunicorn-fastapi-docker)

*Uvicorn과 Gunicorn은 "**웹 서버**" 범주에 속합니다.*
