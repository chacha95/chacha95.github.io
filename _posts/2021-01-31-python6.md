# FastAPI란?

FastAPI는 표준 Python 유형 힌트를 기반으로 Python 3.6 이상으로 API를 빌드하기위한 최신의 빠른 (고성능) 웹 프레임 워크입니다.

주요 기능은 다음과 같습니다.

- 빠름: NodeJS 및 Go와 동등한 매우 높은 성능 (Starlette 및 Pydantic 덕분에). 사용 가능한 가장 빠른 Python 프레임 워크 중 하나입니다.
- 빠른 코드 작성: 기능 개발 속도를 약 200 % ~ 300 % 향상시킵니다. 
-  버그 감소 : 인간 (개발자)이 유발 한 오류의 약 40 %를 줄입니다. *
- 직관적 : 훌륭한 편집기 지원. 어디에서나 완성. 디버깅 시간 단축.
- 쉬움 : 사용하고 배우기 쉽게 설계되었습니다. 문서를 읽는 시간이 줄어 듭니다 
- Short : 코드 중복을 최소화합니다. 각 매개 변수 선언의 여러 기능. 더 적은 버그.
- 견고성 : 프로덕션 준비 코드를 확보하십시오. 자동 대화 형 문서.
- OpenAPI : API에 대한 개방형 표준을 기반으로하며 완전히 호환됩니다 : OpenAPI (이전에는 Swagger라고 함) 및 JSON 스키마.

## ASGI란?

**ASGI**(Asynchronous Server Gateway Interface)는 비동기 가능 Python 웹 서버, 프레임 워크 및 애플리케이션 간의 표준 인터페이스를 제공하기위한 WSGI의 정신적 후속 제품입니다.

WSGI가 동기식 Python 앱에 대한 표준을 제공 한 반면, ASGI는 WSGI 이전 버전과의 호환성 구현과 여러 서버 및 애플리케이션 프레임 워크를 사용하여 비동기 및 동기 앱에 대한 표준을 제공합니다.

- **WebSocket** 지원.
- **GraphQL** 지원
- In-process 백그라운드 작업
- 시작 및 종료 이벤트.
- 요청에 따라 빌드 된 테스트 클라이언트.
- CORS, GZip, 정적 파일, 스트리밍 응답.
- 세션 및 쿠키 지원.
- 하드웨어 종속성 x

## Uvicorn

Uvicorn은 **uvloop** 및 **httptools**를 사용하는 초고속 ASGI 서버

최근까지 Python에는 asyncio 프레임 워크를위한 저수준 서버 / 애플리케이션 인터페이스가 없었습니다.

ASGI는 IO 바인딩 컨텍스트에서 높은 처리량을 달성하는 측면에서 Node 및 Go와 경쟁이 치열한 Python 웹 프레임 워크의 에코 시스템을 지원해야합니다. 또한 WSGI에서 처리 할 수없는 HTTP / 2 및 WebSocket을 지원합니다.

Uvicorn은 현재 HTTP / 1.1 및 WebSocket을 지원합니다. HTTP / 2에 대한 지원이 계획되어 있습니다.

## uvloop

asyncio는 Python 표준 라이브러리와 함께 제공되는 비동기 I/O 프레임 워크입니다.

**asyncio 이벤트 루프를 대체하는 uvloop**

uvloop는 Cython으로 작성되었으며 **libuv** 위에 구축되었습니다.

실제로 nodejs, gevent 및 기타 Python 비동기 프레임 워크보다 2배 이상 빠릅니다. uvloop 기반 asyncio의 성능은 **Go 프로그램의 성능에 가깝습니다.**

**libuv**는 nodejs에서 사용하는 고성능 다중 플랫폼 비동기 I / O 라이브러리입니다.

uvloop는 모든 asyncio 이벤트 루프 API를 구현합니다.

고수준 Python 객체는 저수준 libuv 구조체와 함수를 래핑합니다.

### HTTP 성능 측정

놀랍게도 고성능 HTTP 파서의 도움으로 순수 Python asyncio는 동일한 HTTP 파서를 사용하는 nodejs보다 빠릅니다!

Go는 1KiB 응답의 경우 더 빠르지 만 uvloop + asyncio는 10 / 100KiB 응답의 경우 훨씬 더 좋습니다. 서비스 품질은 Go뿐만 아니라 httptools를 사용하는 asyncio 및 uvloop에서도 우수합니다.

물론 httptools 기반 서버는 매우 최소한이며 다른 구현과 달리 라우팅 논리를 포함하지 않습니다. 그럼에도 불구하고 벤치 마크는 효율적으로 구현 된 프로토콜로 uvloop가 얼마나 빠른지 보여줍니다.

uvloop 및 asyncio는 Python 3.5의 async / await 기능과 결합되어 Python에서 고성능 네트워킹 코드를 그 어느 때보 다 쉽게 작성할 수 있습니다.

## Gunicorn

많은 사람들이 Gunicorn (또는 uWSGI)을 사용하고 있지만 실제로 무엇을하는지 잘 모릅니다.

당신은 그것을 주어진 것으로 받아들이고, 배포 할 때 당신의 WAS(Web Application Server)가 예상대로 작동하기를 바랍니다.

유닉스 접근 방식은 한 가지를 잘 수행하는 작은 도구를 갖는 것입니다.

새로운 웹 애플리케이션을 구축 할 때마다 모든 작업을 직접 수행해야한다고 상상해보십시오.(너무 많은 업무)

- 정적 파일 호스팅
- https 연결 처리
- 충돌에서 복구
- 애플리케이션이 확장 될 수 있는지 확인 (프레임 워크 또는 자체 코드를 통해)

Python 웹 애플리케이션을 프로덕션에 배포 할 때 일반적인 세 가지 구성 요소는 다음과 같습니다.

- Web server (e.g. Nginx)
- WSGI application server (e.g. Gunicorn)
- 실제 application (server) (e.g. Django)

Web server는 요청을 수락하고 도메인를 처리하며 https 연결을 처리합니다.

Application에 도착하기 위한 요청만 애플리케이션 서버와 애플리케이션 자체로 전달됩니다.

Applicatin은 단일 요청을 처리 할 수 있다는 것 외에는 아무것도 신경 쓰지 않습니다.

### Gunicorn is a WSGI server

Gunicorn은 다양한 web server가 상호 작용할 수 있도록 구축되었습니다.

Gunicorn은 웹 서버와 웹 애플리케이션 사이에서 일어나는 모든 일을 처리합니다. 이렇게하면 ㅈ웹 애플리케이션을 코딩 할 때 다음을위한 자체 솔루션을 찾을 필요가 없습니다.

- 여러 웹 서버와 통신
- 한 번에 많은 웹 요청에 반응하고 부하를 분산
- 실행중인 웹 애플리케이션의 여러 프로세스 유지

**WSGI (Web Server Gateway Interface)는 web server와 web application이 서로 통신 할 수 있도록하는 방법입니다.** 즉, 서버 측은 애플리케이션 측에서 제공하는 호출 가능한 객체를 호출합니다.

따라서 애플리케이션 내부 어딘가에 Gunicorn에서 호출 할 수있는 개체가 정의됩니다. 이 개체는 응용 프로그램에 요청 데이터를 전달하고 응답 데이터를 수신하는 데 사용됩니다.

Gunicorn은 웹 애플리케이션의 여러 인스턴스를 실행하여 정상인지 확인하고 필요에 따라 다시 시작하여 해당 인스턴스에 수신 요청을 분산하고 웹 서버와 통신합니다.

## Uvicorn with Gunicorn

Gunicorn은 성숙하고 완전한 기능을 갖춘 **서버** 및 **프로세스** 관리자입니다.

Uvicorn에는 Gunicorn worker class가 포함되어있어 Uvicorn의 모든 성능 이점과 함께 ASGI 애플리케이션을 실행할 수 있으며 Gunicorn의 모든 기능을 갖춘 프로세스 관리를 제공합니다.

이를 통해 즉시 worker 프로세스 수를 늘리거나 줄이고, worker 프로세스를 정상적으로 다시 시작하거나, 다운 타임없이 서버 업그레이드를 수행 할 수 있습니다.

Uvicorn과 Gunicorn은 "**웹 서버**" 범주에 속합니다.

### Uvicorn with Nginx

Nginx는 **reverse proxy server**의 역할을 함

nginx의 uwsgi 모듈을 사용하고 있습니다. Uvicorn은 asgi API를 노출합니다. 따라서 uwsgi 구성 대신 "reverse proxy"구성을 사용해야합니다.

Uvicorn 프로세스 앞에서 Nginx를 프록시로 사용하는 것은 필요하지 않을 수 있지만 추가 복원력을 위해 권장됩니다. Nginx는 정적 미디어를 제공하고 느린 요청을 버퍼링하여 애플리케이션 서버를 가능한 한 많이로드하지 않도록 할 수 있습니다.

Heroku와 같은 관리 환경에서는 서버 프로세스가 이미로드 밸런싱 프록시 뒤에서 실행되고 있으므로 일반적으로 Nginx를 구성 할 필요가 없습니다.

Nginx에서 프록시하는 데 권장되는 구성은 Nginx와 Uvicorn을 실행하는 데 사용되는 프로세스 관리자간에 UNIX 도메인 소켓을 사용하는 것입니다. 이 작업을 수행 할 때 --forwarded-allow-ips = '*'와 함께 Uvicorn을 실행하여 도메인 소켓이 헤더를 프록시 할 소스로 신뢰할 수 있는지 확인해야합니다.

응용 프로그램에 프록시 서버를 연결할 때 응용 프로그램이 들어오는 연결의 클라이언트 주소를 올바르게 결정할 수 있는지 그리고 연결이 http 또는 https를 통해 이루어 졌는지 확인하기 위해 프록시가 헤더를 설정하는지 확인해야합니다.

X-Forwarded-For 및 X-Forwarded-Proto 헤더가 프록시에 의해 설정되고 Uvicorn이 --proxy-headers 설정을 사용하여 실행되는지 확인해야합니다. 이렇게하면 ASGI 범위에 올바른 클라이언트 및 체계 정보가 포함됩니다.

다음은 간단한 Nginx 구성의 모습입니다. 이 예에는 프록시 헤더 설정 및 UNIX 도메인 소켓을 사용하여 애플리케이션 서버와 통신하는 방법이 포함됩니다.

## Starlette

Starlette는 경량 ASGI 프레임 워크 / 툴킷으로 고성능 비동기 서비스를 구축하는 데 이상적

Python 및 Flask와 같은 생태계 내의 WSGI 웹 서버 프레임 워크는 간단한 동기식 API를 구축하는 데까지 충분한 작업을 수행하는 데 충분합니다. 하지만 앱의 백엔드를 확장해야한다면 어떨까요?

예를 들어 계산 예측을 확장 할 수있는 기계 학습 모델이 있다면 어떨까요? 또한 모델 위에있는 API 계층도 확장 할 수 있는지 확인해야합니다. 이러한 경우, 동기식 WSGI 프레임 워크로는 충분하지 않을 수 있습니다. Starlette를 구출하세요!

Starlette는 완전히 비동기 적으로 실행할 수있는 ASGI 웹 서버 프레임 워크입니다. 따라서 Starlette는 대규모 요청을 처리하여 위에서 언급 한 문제를 해결할 수 있습니다! 앞으로 나올 섹션에서는 다음 내용을 배우게됩니다.

## Pydantic

Python기반 data validataion

pydantic은 런타임에 type hinting을 적용하고 data validation 실행

pycharm에서 pydantic plugin 지원

pydantic을 이용해 FastAPI server에서 post method의 body를 받을 수 있음

# Basic

```python
from datetime import datetime
from typing import List, Optional
from pydantic import BaseModel

class User(BaseModel):
    id: int
    name = 'John Doe'
    signup_ts: Optional[datetime] = None
    friends: List[int] = []

external_data = {
    'id': '123',
    'signup_ts': '2019-06-01 12:22',
    'friends': [1, 2, '3'],
}
```
