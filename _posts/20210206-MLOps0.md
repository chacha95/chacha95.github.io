# MLOps란?

MLOps는 AI의 상업적 사용 자체가 상당히 새로운 것이기 때문에 비교적 새로운 분야입니다.

MLOps는 데이터 과학자와 앱 개발자가 ML 모델을 프로덕션으로 가져올 수 있도록 지원합니다. MLOps를 사용하면 ML 수명주기의 모든 자산을 추적 / 버전 화 / 감사 / 인증 / 재사용 할 수 있으며이 수명주기 관리를 간소화하는 오케스트레이션 서비스를 제공합니다.

## MLOps와 DevOps 차이

이러한 모든 차이점의 원인은 머신 러닝이 무엇이며 어떻게 실행되는지에서 찾을 수 있습니다. 소프트웨어는 입력에 대한 응답으로 작업을 수행하며이 ML 및 주류 프로그래밍은 유사합니다. 그러나 행동이 성문화되는 방식은 크게 다릅니다.

기존 소프트웨어는 작업을 명시적인 규칙으로 코드화합니다. 가장 간단한 프로그래밍 예제는 프로그램이 'hello world'를 출력해야한다고 단순히 코드화 한 'hello world'프로그램 인 경향이 있습니다. 그런 다음 추가 제어 구조를 추가하여 입력에 대한 응답으로 작업을 수행하는 더 복잡한 방법을 추가 할 수 있습니다. 더 많은 제어 구조를 추가할수록 프로그래밍 언어를 더 많이 배웁니다. 이 규칙 기반 입력-출력 패턴은 입력이 모두 키보드를 통해 이루어지고 출력이 거의 모든 텍스트 인 구형 터미널 시스템과 관련하여 이해하기 쉽습니다. 그러나 입력 및 출력 유형이 매우 다양하고 복잡 할 수 있지만 우리가 상호 작용하는 대부분의 소프트웨어에서도 마찬가지입니다.

ML은 명시 적으로 코드화하지 않습니다. 대신 규칙은 데이터에서 패턴을 캡처하여 간접적으로 설정됩니다. 따라서 ML은 수치 적으로 처리 할 수있는보다 집중된 유형의 문제에 더 적합합니다. 예를 들어, 경험, 교육, 위치 등과 같은 데이터 포인트 / 특징에서 급여를 예측합니다. 이것은 회귀 문제의 경우입니다. 여기서 목적은 다른 변수의 값에서 변수 (급여)의 값을 사용하여 예측하는 것입니다. 이전 데이터의. 기계 학습은 또한 변수 값을 예측하는 대신 모델이 데이터 포인트가 특정 클래스에 속할 확률을 출력하는 분류 문제에도 사용됩니다. 분류 문제의 예는 다음과 같습니다.

    숫자에 대해 손으로 쓴 샘플이 주어지면 어떤 숫자가 어느 것인지 예측하십시오.
    카테고리에 따라 사물의 이미지를 분류합니다. 꽃의 종류

여기에서 ML이 수행되는 방법에 대한 모든 세부 정보를 이해할 필요는 없습니다. 그러나 ML 모델이 학습되는 방식을 파악하면 도움이됩니다. 따라서 경험, 교육, 위치 등에 대한 데이터에서 급여를 예측하는 것과 같은 회귀 문제와 관련된 문제를 높은 수준에서 고려해 보겠습니다. 이것은 데이터 포인트를 통해 프로그래밍 방식으로 선을 그려서 해결할 수 있습니다.

MLOps는 DevOps와 어떻게 다릅니 까?

     데이터 / 모델 버전 관리! = 코드 버전 관리-스키마 및 원본 데이터 변경에 따라 데이터 세트의 버전을 지정하는 방법
     코드 + (잠재적으로 고객) 데이터를 처리 할 때 디지털 감사 추적 요구 사항 변경
     모델 재사용은 입력 데이터 / 시나리오를 기반으로 모델을 조정해야하므로 소프트웨어 재사용과 다릅니다.
     모델을 재사용하려면 모델에 대한 학습을 미세 조정 / 전송해야 할 수 있습니다 (학습 파이프 라인이 필요함을 의미).
     모델은 시간이 지남에 따라 쇠퇴하는 경향이 있으며 프로덕션 컨텍스트에서 유용하게 유지되도록 요청시 모델을 재교육 할 수있는 기능이 필요합니다.

## **MLOps: Taking Enterprise AI Mainstream**

AI의 빅뱅은 2012 년 한 연구원이 딥 러닝을 사용한 이미지 인식 대회(2012년)에서 우승했을 때 울려 퍼졌습니다. 

오늘날 AI는 웹 페이지를 번역하고 고객 서비스 호출을 자동으로 라우팅합니다. 병원에서 X- 레이를 읽고 은행에서 신용 위험을 계산하며 소매 업체의 재고 선반을 통해 판매를 최적화하는 데 도움이됩니다.

MLOps는 엔터프라이즈 애플리케이션을 효율적으로 작성, 배포 및 실행하는 현대적인 관행 인 DevOps의 기존 분야를 모델로합니다. DevOps는 10 년 전 소프트웨어 개발자 (Devs) 부족과 IT 운영 팀 (Ops)이 협력 할 수있는 방법으로 시작되었습니다.

MLOps는 데이터 과학자를 팀에 추가하여 데이터 세트를 큐레이팅하고이를 분석하는 AI 모델을 구축합니다. 여기에는 체계적이고 자동화 된 방식으로 모델을 통해 데이터 세트를 실행하는 ML 엔지니어도 포함됩니다.

이는 원시 성능과 관리 엄격 성면에서 큰 도전입니다. 데이터 세트는 방대하고 증가하며 실시간으로 변경 될 수 있습니다. AI 모델은 실험, 튜닝 및 재교육의주기를 통해주의 깊게 추적해야합니다.

## Lifecycle Tracking for Data Scientists

하나의 프로세스로 엮어내는 일련의 관련 작업입니다.

데이터 과학자는 외부 소스 및 내부 데이터 레이크에서 데이터 세트를 자유롭게 잘라내어 붙여 넣을 수 있어야합니다. 하지만 이들의 작업과 데이터 세트는 신중하게 레이블을 지정하고 추적해야합니다.

마찬가지로, 그들은 당면한 작업에 잘 맞는 훌륭한 모델을 만들기 위해 실험하고 반복해야합니다. 따라서 유연한 샌드 박스와 견고한 저장소가 필요합니다.

또한 프로토 타입, 테스트 및 프로덕션을 통해 데이터 세트와 모델을 실행하는 ML 엔지니어와 협력 할 방법이 필요합니다. 모델을 쉽게 해석하고 재현 할 수 있도록 자동화와 세부 사항에 대한주의가 필요한 프로세스입니다.

오늘날 이러한 기능은 클라우드 컴퓨팅 서비스의 일부로 제공되고 있습니다. 머신 러닝을 전략적으로 보는 기업은 성장하는 공급 업체의 MLOps 서비스 또는 도구를 사용하여 자체 AI 센터를 구축하고 있습니다.
